Gulp the whole thing

Jurvis had done a decent bit of front-end web development before, and he decided to place the Electron app’s JS and CSS on Gulp processes. Gulp is a framework that lets you place your CSS and JS through a build process.

SASS

SASS is a CSS preprocessor, it brings in extra functionality (such as writing variables) in your SASS files (which are special CSS files). During the build process, SASS will fill in the variables and spit out the “compiled” file, ready for use in the browser (or Electron’s Chromium, in this case). It lets you maintain cleaner CSS while making its “compiled” product still usable and valid.

Babel

Babel transpires ES6 JS code into ES5 JS code. The Chromium in Electron does not seem to support ES6 outside of the main process and this step is necessary for us to maintain an ES6 codebase throughout.

Bring in the ES6 goodness

The Javascript language specification introduced a new version called ECMAScript 6 (aka ECMAScript 2015). It brings in a much improved syntax, which I’ll show below:

Function declarations

// ES5 function
function hi() {
    console.log(‘bye’);
}

// ES6 function
let hi = () => {
    console.log(‘bye’);
}

String concatenation

var number = 9;

// ES5
var str = ‘The number is ‘ + 9;

// ES6
let str = `The number is ${number}`;

For-of loop

ES5 and before hasn’t provided good ways to iterate over data structures. ES6 introduces the new for-of loop!

var array = [‘apple’, ‘banana’, ‘chocolate’];
// This loop came from the C family of languages
for (var i = 0; i < array.length; i++) {
    console.log(array[i]); // Prints out “apple”, “banana”, “chocolate”
}

// This loop doesn’t do anything particularly useful
for (var item in array) {
    console.log(item); // Prints out “0”, “1”, “2”, rather useless.
}

// This loop was introduced in ES5. It has no way of letting you break the loop in between.
array.forEach(function(item) {
    console.log(item); // Prints out “apple”, “banana”, “chocolate”
});

// ES6 for-of loop. Finally, we have a nice, clean and fully functional for loop!
for (var item of array) {
    console.log(item); // Prints out “apple”, “banana”, “chocolate”
}

There are more features which help to maintain a cleaner codebase, such as classes and let/const declarations. The addition of ES6 has really helped with maintaining the code 😄.

ESLint

This is a really helpful tool for helping you maintain clean JS code. It checks for many things (such as unused variables and misplaced indentation). It works as a plugin for your text editor (Atom, Sublime Text and VS Code all have it) and it lints your code according to a list of rules defined in .eslintrc.
